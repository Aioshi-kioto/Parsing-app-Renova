# Использование

Два скрипта работают по одному принципу: **готовый URL с фильтрами**. Настраиваешь фильтры в браузере Zillow, копируешь URL — скрипт парсит.

---

## 1. parse_from_url.py — универсальный парсер

**Любой URL, любая область, любые фильтры.**

### Как использовать

1. Откройте Zillow в обычном браузере
2. Настройте фильтры:
   - Listing Status: Sold
   - Price: Min $600,000
   - Home Type: Houses
   - Basement: Unfinished
   - Sold in last: 36 months
3. Настройте область на карте (zoom, границы)
4. Скопируйте URL из адресной строки
5. Запустите скрипт и вставьте URL
6. Пройдите капчу вручную (если появится)
7. Нажмите Enter в консоли

### Запуск

```bash
python scripts/parsers/parse_from_url.py
```

### Результат

- **< 500 результатов** — сохраняет сразу в `results_from_url.json`
- **≥ 500 результатов** — QuadTree разбиение, checkpoint для продолжения после ошибок

---

## 2. parse_seattle_urls.py — Seattle по готовым URL

**4 предзаданных URL для Seattle** (Sold, $550k+, basement unfinished, 36 months).

### Как использовать

1. Запустите скрипт
2. Нажмите Enter
3. Пройдите капчу на первой части
4. Дождитесь завершения

### Запуск

```bash
python scripts/parsers/parse_seattle_urls.py
```

### Результат

- `seattle_urls_results.json` — все дома Seattle
- Метаданные: время парсинга, количество частей

---

## Общее

- **Капча** — при первом запросе пройдите вручную, скрипт продолжит
- **Checkpoint** — при QuadTree прогресс сохраняется в `scraping_checkpoint_url.json`, можно перезапустить
- **Логи** — в папке `logs/`
